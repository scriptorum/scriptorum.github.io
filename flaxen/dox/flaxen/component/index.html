package flaxen.component

ActionQueue

In an entity-component-system, systems represent the behavior. Systems examine the state of the game via the entities and their components, and react to it, by adding/removing entities and modifying components. This in turn can trigger another reaction, which triggers another. While systems can be used for complicated chain of actions and reactions, sometimes it can make the design needlessly complicated.

DeathBox

A death box is an area on the screen for automatically removing entities that either stray out of or into specified areas. This is a very basic collision box implementation. Only the registration point of the entity will be checked against the box. You may share this component between several entities, and you may dynamically alter the death box.

Intent

Intents are a way to decouple input from reaction to that input. When the appropriate input is detected, instead of taking that action, create an Intent. Then in a subsequent system process that Intent. This has two effects. First, it simplifies the InputSystem so it no longer needs to know about operational details of the reaction. Second, it allows a third-party system to introduce its own intents and drive reactions without player inputs.
